# Instalación en un servidor nuevo (Windows)

Servicio de Windows (.NET 8) que lee documentos de SAP B1 vía Service Layer, genera y firma
el XML SIFEN, lo envía a la SET, y encola/envía los correos con PDF (Crystal Reports) y XML.

---

## 1. Requisitos en el servidor nuevo

| Componente | Para qué | Nota |
|---|---|---|
| **Windows Server** (2016+) | host del servicio | |
| **.NET 8 Runtime** (o publicar self-contained) | ejecutar el .exe | https://dotnet.microsoft.com/download/dotnet/8.0 → "ASP.NET Core / .NET Runtime x64" |
| **SAP HANA ODBC Client 64-bit** (`HDBODBC`) | conexión del servicio a HANA | viene con SAP HANA Client. Sin esto, `Driver=HDBODBC` falla |
| **SAP HANA ODBC Client 32-bit** (`libodbcHDB32`) | Crystal Reports corre en 32 bits | instalar el HANA Client 32-bit también |
| **SAP Crystal Reports Runtime 32-bit** | generar el PDF de la factura | los assemblies deben quedar en `C:\Windows\assembly\GAC_MSIL\CrystalDecisions.*` |
| **PowerShell 32-bit** | `SysWOW64\WindowsPowerShell\v1.0\powershell.exe` (ya viene con Windows) | |
| **Acceso de red** | Service Layer SAP (`:50000`), HANA (`:3xx15`), SIFEN (`sifen.set.gov.py`), SMTP | abrir firewall de salida |

> El servicio **no** necesita SAP B1 client instalado, pero sí que el **Service Layer** esté
> accesible desde este servidor.

## 2. Publicar la aplicación

Desde tu máquina de desarrollo (en la raíz del repo):

```bash
dotnet publish src/src.csproj -c Release -r win-x64 --self-contained false -o ./publish-win
```

Si no querés instalar el runtime .NET en el servidor, usá self-contained:

```bash
dotnet publish src/src.csproj -c Release -r win-x64 --self-contained true -p:PublishSingleFile=false -o ./publish-win
```

Copiá **todo** el contenido de `publish-win` al servidor.

## 3. Estructura de carpetas en el disco C

```
C:\SifenService\
├── src.exe                      <- ejecutable (viene del publish)
├── *.dll                        <- dependencias
├── src.runtimeconfig.json
├── config\
│   └── config.json              <- CREAR A MANO (no está en git)
├── XML\                         <- se crea sola: XML firmados por empresa
│   └── <CompanyDB>\
├── BackupXml\                   <- se crea sola
└── logs\                        <- se crea sola: sifen-service-YYYYMMDD.log
```

Solo tenés que crear `C:\SifenService` y `C:\SifenService\config`. Las carpetas `XML`,
`BackupXml` y `logs` las crea el servicio al arrancar (todas se resuelven relativas al
directorio del .exe, por eso no hay rutas hardcodeadas que tocar).

```powershell
New-Item -ItemType Directory -Path C:\SifenService\config -Force
```

## 4. `config/config.json`

Archivo obligatorio; si falta, el servicio muere al arrancar y escribe `error.log`.

```json
{
  "SapServiceLayerList": [
    {
      "Url": "https://sap-server:50000/b1s/v1/",
      "CompanyDB": "NOMBRE_EMPRESA_1",
      "UserName": "manager",
      "Password": "xxxxx",
      "Sifen": {
        "Url": "https://sifen-test.set.gov.py/de/ws/sync/recibe.wsdl",
        "IdCSC": "0001",
        "CSC": "ABCD0000000000000000000000000000"
      }
    }
  ],
  "HanaDatabase": {
    "ServerNode": "hana-server:30015",
    "UserName": "SYSTEM",
    "Password": "xxxxx",
    "Schema": "SAP_SIFEN",
    "OdbcDsn": "HANA_FE"
  }
}
```

- `SapServiceLayerList` es una **lista**: agregá un bloque por cada empresa (multi-empresa).
- `Sifen.Url`: usar el endpoint de **test** primero, luego el de producción.
- La cadena de conexión se arma como `Driver=HDBODBC;ServerNode=...;UID=...;PWD=...`.

## 5. DSN ODBC 32-bit por empresa (necesario para los PDF)

Sin esto los PDF salen **en blanco**. Por cada `CompanyDB`:

1. Ejecutar `C:\Windows\SysWOW64\odbcad32.exe` (el de 32 bits, no el de Panel de Control).
2. Pestaña **System DSN** → **Add** → driver **HDBODBC32 / SAP HANA**.
3. **Nombre del DSN = exactamente el CompanyDB** (ej. `CONJUNTO_ALTAVIDALUQUE`).
4. Server: `hana-server:30015`.
5. En **Settings → Special property settings** agregar `CURRENTSCHEMA = <CompanyDB>`.

Verificable después en `HKLM\SOFTWARE\WOW6432Node\ODBC\ODBC.INI\<DSN>`.

También conviene crear el DSN **64-bit** (`odbcad32.exe` normal) si querés probar la conexión
del servicio, aunque el servicio usa cadena directa sin DSN.

## 6. Objetos en HANA (esquema `SAP_SIFEN`)

El servicio **no crea las tablas**: hay que crearlas antes.

```sql
CREATE SCHEMA SAP_SIFEN;

CREATE COLUMN TABLE SAP_SIFEN.DOCUMENTOS_SIFEN (
  ID                 BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  BASE_DATOS         NVARCHAR(100),
  DID                NVARCHAR(50),
  LOTE               NVARCHAR(50),
  CDC                NVARCHAR(50),
  XML                NCLOB,
  ESTADO             NVARCHAR(50),
  DCODRES            NVARCHAR(20),
  TIPO_DOCUMENTO     NVARCHAR(50),
  SERVICIO           NVARCHAR(50),
  FECHA_CREACION     TIMESTAMP,
  FECHA_ENVIO        TIMESTAMP,
  FECHA_RESPUESTA    TIMESTAMP,
  MENSAJE_RESPUESTA  NVARCHAR(2000),
  RESPUESTA_BRUTA    NCLOB
);

CREATE COLUMN TABLE SAP_SIFEN.COLA_EMAILS (
  ID                BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  BASE_DATOS        NVARCHAR(100),
  CDC               NVARCHAR(50),
  DOC_ENTRY         INTEGER,
  TIPO_DOCUMENTO    NVARCHAR(50),
  EMAIL_RECEPTOR    NVARCHAR(200),
  NOMBRE_RECEPTOR   NVARCHAR(200),
  QR                NCLOB,
  RUTA_XML          NVARCHAR(500),
  ESTADO            NVARCHAR(20),      -- PENDIENTE / ENVIADO / ERROR / SIN_EMAIL
  INTENTOS          INTEGER DEFAULT 0,
  ULTIMO_ERROR      NVARCHAR(2000),
  FECHA_CREACION    TIMESTAMP,
  FECHA_ENVIO       TIMESTAMP,
  SAP_ACTUALIZADO   NVARCHAR(2),
  SAP_INTENTOS      INTEGER DEFAULT 0,
  SAP_ULTIMO_ERROR  NVARCHAR(500),
  SAP_FECHA_UPDATE  TIMESTAMP
);
```

El usuario HANA del `config.json` necesita `SELECT/INSERT/UPDATE` sobre este esquema y
`SELECT` sobre los esquemas de cada empresa.

## 7. Objetos en SAP B1 (UDT/UDF) — por cada empresa

El servicio los consulta por Service Layer, así que deben existir en **cada** `CompanyDB`:

- **`U_CERTIFICADOS`** (UDT) — campos `U_ARCHIVO` (certificado .pfx en base64), `U_PWD`
  (contraseña), `U_ACTIVO` (`Y`/`N`). El servicio toma el primero con `U_ACTIVO = 'Y'`.
- **`U_CONFIG_EMAIL`** (UDT) — `U_BASE_DATOS`, `U_CORREO`, `U_SMTP`, `U_PUERTO`, `U_PWD`
  (encriptada), `U_SSL`, `U_TPL_FAC`, `U_TPL_NC`, `U_RPT_FAC`, `U_RPT_NC`.
  Los `U_RPT_*` son **rutas a los .rpt** — deben ser accesibles desde este servidor
  (ruta local o UNC con permisos para la cuenta del servicio).
- **UDFs en documentos**: `U_EXX_FE_CDC`, `U_TIM`, y en el maestro de empresa/sucursal
  `U_NDEP`, `U_NCIU`, `U_NLOC`, `U_FECHAV`.

Si migrás desde el servidor viejo, exportá estas UDT/UDF y los `.rpt` tal cual.

## 8. Registrar el servicio de Windows

```powershell
sc.exe create SifenService `
  binPath= "C:\SifenService\src.exe" `
  DisplayName= "Servicio SIFEN SAP" `
  start= auto

sc.exe description SifenService "Facturacion electronica SIFEN - SAP Business One"
sc.exe failure SifenService reset= 86400 actions= restart/60000/restart/60000/restart/60000

sc.exe start SifenService
```

**Importante — cuenta del servicio:** si los `.rpt` están en una ruta de red, o el DSN debe
ser visible, corré el servicio con una cuenta de dominio en vez de `LocalSystem`:

```powershell
sc.exe config SifenService obj= "DOMINIO\usuario" password= "xxxxx"
```

Esa cuenta necesita "Log on as a service" y permiso de escritura sobre `C:\SifenService`.

## 9. Verificación

1. **Probar en consola primero** (antes de registrar el servicio):
   ```powershell
   cd C:\SifenService
   .\src.exe
   ```
   Deberías ver `MultiBaseSAPCDCService iniciado...` y `Procesando base de datos: <CompanyDB>`.
   Si falla, mirá `C:\SifenService\error.log`.

2. **Logs**: `C:\SifenService\logs\sifen-service-<fecha>.log`.

3. **XML generados**: `C:\SifenService\XML\<CompanyDB>\Documento_<CDC>.xml`.

4. **Prueba de PDF**: emitir una factura de prueba y verificar que el PDF adjunto no venga
   vacío. Si viene vacío → falta el DSN 32-bit con `CURRENTSCHEMA` (paso 5).

5. **Cola de correos**:
   ```sql
   SELECT ESTADO, COUNT(*) FROM SAP_SIFEN.COLA_EMAILS GROUP BY ESTADO;
   ```
